# kFarmAI 2.0 — AI 세션 컨텍스트

## 프로젝트 정의
- 이름: kFarmAI 2.0
- URL: kfarmai.com / kfarmai.kr
- 성격: 개인 사이드 프로젝트 (회사 업무와 완전 분리)
- 목표: 농업·식물 문제 해결 AI 커뮤니티 플랫폼

## 기술 스택
- Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS
- DB/Auth: Supabase (PostgreSQL + RLS)
- AI: Claude API (claude-sonnet 단일)
- 지도: Kakao Maps API
- 배포: Vercel
- 스토리지: Supabase Storage

## 화면 구조
모바일 메인 = 스와이프 3페이지
- 페이지1: 커뮤니티 (식물 병원, 질문방, 작물상담방, 사랑방)
- 페이지2: AI 진단 (사진 업로드 → Claude API → 처방)
- 페이지3: 농자재·지역 (전국 농약사 지도, DB, 보조사업)

## 핵심 설계 원칙
1. 모듈 독립성: community / diagnosis / agridata 직접 의존 금지
2. 단일 Claude API (3단계 파이프라인 없음)
3. 커뮤니티 우선, 농자재 후순위
4. 트래픽 먼저, 수익화 나중
5. 보안: RLS 전 테이블, 환경변수 Git 커밋 금지

## 현재 kfarmai.com 자산 (유지 필수)
- 전국 농약사 3,019개소 (agri_map.html, map.html)
- 상토 15사, 종자 57사, 작물보호제 39사, 비료 8사
- 보조사업 아카이브 (news_sub.html)
- 기존 URL 구조 유지 (SEO 손실 방지)

## 처방 구조
- 일반 처방: 무료 공개 (관리 방향, 원인 후보)
- 유료회원사: 제품 코드, 링크, 가까운 농약사 연결

## 커뮤니티 게시판 (Phase 1 MVP)
1. 식물 병원 (사진 진단 허브)
2. 식물 질문방 (홈가드닝)
3. 작물 상담방 (전문농업인)
4. 내 식물 자랑
5. 농부 사랑방

## Phase 1 완료 목표
